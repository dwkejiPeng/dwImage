# Security Policy

## Supported Versions

This project is under active development. Security fixes are generally applied to the latest code in the default branch.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for sensitive security problems.

Instead, report security issues privately to the maintainer through your preferred private contact channel before public disclosure.

Please include:

- A clear description of the issue
- Affected area
- Reproduction steps if possible
- Potential impact

You can expect:

- Acknowledgement after review
- Validation or follow-up questions if needed
- A fix plan when the issue is confirmed
